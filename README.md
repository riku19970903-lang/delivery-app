# 配送スマイル管理 MVP

配送業の管理者とメンバーが、出退勤・点呼・配達個数・報酬見込み・休み希望・シフト・車両不具合をローカルで管理できる FastAPI + SQLite 製のMVPです。

## 起動方法

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
uvicorn app:app --reload
```

ブラウザで `http://127.0.0.1:8000` を開きます。

## スマホからアクセスする起動方法

PCとスマホを同じWi-Fiに接続し、PC側で以下を実行します。

```powershell
.\start_lan.ps1
```

この起動方法では `0.0.0.0:8020` で待ち受けるため、スマホから以下の形式でアクセスできます。

```text
http://<PCのIPv4アドレス>:8020/login
```

このPCでは、確認時点のWi-Fi側IPv4アドレスは `192.168.210.164` でした。

```text
http://192.168.210.164:8020/login
```

スマホから開けない場合は、Windows Defender ファイアウォールの確認画面で Python / uvicorn の通信を許可してください。

## 画面ごとの説明

### 管理者

- 管理者ダッシュボード: 本日の点呼件数、配達入力、車両不具合の概要を確認
- メンバー管理: 登録、編集、停止
- 単価・車両貸出設定: メンバーごとの単価、車両貸出タイプ、日貸し料金、月貸し料金を確認・変更。管理者専用で、他メンバーには表示しません
- 配達エリア管理: 地区と町会を追加、編集、削除。町会は地区に紐づきます
- 安全記録一覧: 点呼、酒気帯び確認、健康状態、管理者確認欄を確認。対象年月を選んで印刷用画面を開けます
- 配達一覧: 日別の配達個数を確認
- 売上・報酬: メンバーごとの日別・月別報酬見込みを確認
- 休み希望一覧: メンバーからの希望を確認
- シフト作成: カレンダーの日付をクリックし、メンバー、出勤区分、町会・配達エリアだけを選んで保存
- 車両不具合一覧: メンバー入力の不具合を確認

### メンバー

- ホーム: 自分専用の当日サマリーと自動コメントを表示
- 出退勤・点呼: 出勤時・退勤時の安全確認を入力
- 配達個数入力: 配達完了、転送、夜間、集荷、大型、車両貸出を入力
- 報酬確認: 自分の単価に基づく日別・月別報酬見込みを確認
- 休み希望: 来月の休み希望を入力
- シフト確認: 自分のシフトだけ確認。日付、出勤区分、町会・配達エリアを表示
- 車両不具合: 自分の車両の不具合を報告

## 報酬計算

日別報酬は以下で計算します。

```text
配達完了個数 × 配達単価
+ 転送個数 × 転送単価
+ 夜間個数 × 夜間料金
+ 集荷個数 × 集荷料金
+ 大型個数 × 大型料金
- 車両貸出料金
```

車両貸出は管理者がメンバーごとに設定します。日貸しはその日の報酬から日貸し料金を差し引き、月貸しは月別報酬計算時に月1回だけ月貸し料金を差し引きます。メンバー同士で単価や車両料金は見えません。

## シフト作成の使い方

管理者でログインし、`シフト` を開きます。

1. カレンダーの日付をクリック
2. メンバー名を選択
3. 出勤区分を `出勤`、`1便`、`2便`、`3便`、`休み` から選択
4. 休み以外の場合は、担当する町会・配達エリアをチェックボックスで複数選択
5. `決定して保存` を押す

休みを選んだ場合、町会・配達エリア欄は非表示になり、町会未選択で保存できます。

メンバーの `シフト確認` 画面では、自分のシフトだけがカレンダー形式で表示されます。休みの日は `休み` のみ表示されます。

カレンダー上では町会名を短く表示します。例: `高松1丁目、高松2丁目、高松3丁目` は `高1.2.3`、`土支田2丁目、土支田3丁目` は `土2.3` と表示します。詳細や一覧では正式名称も確認できます。

## メンバー削除

管理者の `メンバー` 画面では、在籍中メンバーと退職済みメンバーを分けて表示します。

`削除（退職扱い）` を押すと確認メッセージが表示され、実行するとメンバーは無効化されてログインできなくなります。完全削除ではないため、過去の配達記録、点呼記録、安全記録、報酬記録、シフト記録は残ります。

退職済みメンバーには `完全削除` ボタンが表示されます。実行前に最終確認が出ます。完全削除は過去記録を壊さないため、ユーザー行を物理削除せず、名前・ログインID・電話・車両などの個人情報を匿名化して退職済み一覧から非表示にします。匿名化後は元のログインIDではログインできません。

## 安全記録PDF出力

管理者でログインし、`安全記録` 画面から対象年月とメンバー名を選んで `運転日報PDF出力` を押します。

印刷用画面が開くので、ブラウザの印刷機能で `PDFとして保存` を選ぶと、A4横向きの運転日報として保存できます。1ヶ月分を日付ごとに1行で表示し、2ページ目以降も表ヘッダーが出るようにしています。

運転日報の列は、日付、車両番号、車両状態、点呼、アルコール確認、開始時刻、終了時刻、開始地点、終了地点、走行距離、経由地点、備考です。未入力の項目は空欄で出力されます。

## 休み希望カレンダー

管理者の `休み希望` 画面は月表示カレンダーです。休み希望がある日にはメンバー名だけ表示し、名前を押すと理由を確認できます。日曜と祝日は赤系、土曜は青系で表示します。

## 車両不具合の状態管理

管理者は `車両不具合` 画面で状態を変更できます。状態は `未対応`、`確認中`、`修理依頼中`、`対応済`、`修理済` です。変更履歴は不具合ごとに残り、メンバーは自分が登録した不具合の現在状態を確認できます。

## 点検表アップロード

メンバーは `点検表` 画面から、スマホで撮影した点検表画像をアップロードできます。日付は初期値として今日が入り、メンバー名はログイン情報から自動表示されます。

画像は `uploads/inspection_sheets/` に保存し、DBの `inspection_sheets` に日付、メンバー、ファイルパス、OCR状態を保存します。同じメンバー・同じ日付で再アップロードした場合は画像を差し替えます。

管理者は `点検表` 画面で日付・メンバー別に提出状況を確認できます。未提出メンバーも表示され、画像を押すと拡大表示できます。将来OCRで数字を読み取れるよう、`ocr_status` と `ocr_text` を用意しています。

配達個数入力画面からも点検表画像を任意でアップロードできます。画像は `uploads/inspection_slips/` に保存され、配達入力ID、メンバー、日付に紐づきます。管理者の配達一覧では `点検表あり` / `点検表なし` とサムネイルを確認できます。

## 安全管理・法令対応の考え方

警察庁の安全運転管理者制度、国土交通省の自動車運送事業におけるアルコール検知器使用に関する公開情報を参考に、運転前後の酒気帯び確認、アルコール検知器使用有無、酒気帯び有無、顔色・呼気・声の調子、体調、備考、管理者確認欄を記録できる形にしています。

法令の最終判断や帳票要件は、社労士・行政書士・運行管理者など専門家に確認してください。
